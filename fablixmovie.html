Fablix Movie DB Web App

Oct 24, 2024 • Java • MySQL • Tomcat • REST APIs • jQuery • AWS

A full-stack movie database with secure login, full-text search, and auto-complete—optimized on AWS for thousands of titles. Github: https://github.com/vctruong100/fablixmovie

Overview

Developed as a course project to demonstrate secure login flows, full-text search, and performance optimization in a dynamic web application.

Problem

Users needed a secure, responsive way to search and browse a large movie dataset with filtering and fast search feedback.

My Role

  • Full-stack Developer
  • Deployed AWS EC2/Tomcat & RDS MySQL
  • Implemented SHA-256 encryption & session management

Key Deliverables

  • Secure login & role-based access control
  • Full-text search with auto-complete
  • High-performance indexing (>10k records)

Page Breakdown

Login Page

The login form features labeled inputs, client-side validation, and an inline reCAPTCHA to prevent bots. Focus states highlight fields, and error messages appear beneath inputs on invalid submissions.

Home/Search Page

Users can filter movies by title, year, director, or star. The persistent nav bar stays in view, and AJAX-powered genre/letter lists load instantly—no full-page reload required.

Movie List Page

This page displays results from the search or category queries. Users can sort movies by title and rating in ascending or descending order, control pagination, and select how many results to display per page.

Movie Details Page

Displays detailed metadata about the selected film including year, director, rating, genres, and associated stars. Includes an "Add to Cart" button and a smooth navigation experience back to search results.

Shopping Cart Page

Users can view their selected movies, update quantities, or remove items. The total cost is dynamically calculated. A clear “Proceed to Payment” button directs users to finalize their purchase.

Payment Page

Provides a clean form for entering billing details including name, credit card, and expiration date. Data is securely validated before submission, and errors are handled with clear feedback.

Lessons & Next Steps

  • Optimized indexing reduced average search time from 250ms to 80ms.
  • Planned to integrate Elasticsearch for advanced filtering.
  • Next phase: mobile-responsive redesign & containerized deployment with Docker.
